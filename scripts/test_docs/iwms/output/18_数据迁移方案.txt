IWMS 数据迁移方案
==================

文档信息
--------
作者: 赵雪 (DBA)
日期: 2026年5月15日
版本: v1.1

一、迁移概述
------------

将速达物流集团现有仓储管理系统（Delphi + SQL Server 2008）中的历史数据
迁移至新系统 IWMS（PostgreSQL 16）。

数据迁移范围：
- 商品主数据：约 12,000 条 SKU
- 库存数据：约 50,000 条实时库存记录
- 历史出入库单：约 300,000 条（含明细）
- 用户及权限数据：约 200 个用户，8 个角色
- 供应商和客户数据：约 3,500 条

二、数据清洗规则
----------------

1. 空值处理
   - 商品名称（goods_name）为空 → 填入 "UNKNOWN_SKU_" + 原系统ID
   - 供应商联系电话为空 → 填入 "000-00000000"
   - 货位编码为空 → 标记为 "ZONE-TEMP" 待后续人工分配

2. 编码转换
   - 原系统使用 GB2312 → 迁移后统一 UTF-8
   - 原系统品名字段存在特殊字符（如 ℃、φ）→ 需要人工确认后替换
   - 日期格式：原系统 "MM/DD/YYYY" → 新系统 "YYYY-MM-DD"

3. 数据去重
   - 商品表：以 (goods_code, supplier_id) 为唯一键，重复记录保留最新
   - 出入库单号：全局唯一校验，重复单号追加 "-DUP" 后缀并标记

4. 非法数据
   - 库存负数 → 一律置为 0，记录异常日志供人工复核
   - 出库明细中的 SKU 在商品表不存在 → 标记为孤立记录，单独输出报告
   - 审批人 ID 不在用户表中 → 替换为 "SYSTEM_AUTO"

三、表映射关系
--------------

| 旧表 (SQL Server) | 新表 (PostgreSQL) | 说明 |
| T_Goods | inventory (sku_id) | 商品主数据 |
| T_Stock | inventory (quantity) | 库存数量 |
| T_In_Master | inbound_orders | 入库单主表 |
| T_In_Detail | inbound_items | 入库明细 |
| T_Out_Master | outbound_orders | 出库单主表 |
| T_Out_Detail | outbound_items | 出库明细 |
| T_Users | users | 用户表，密码需全部重置 |
| T_Roles | roles | 角色表 |
| T_Supplier | suppliers | 供应商 |
| T_Customer | customers | 客户 |

四、迁移步骤
------------

1. 全量导出：使用 SSIS 将旧系统数据导出为 CSV（UTF-8 编码）
2. 数据校验：运行校验脚本 check_migration_data.py 检查数据完整性和格式
3. 数据清洗：运行 clean_migration_data.py 执行上述清洗规则
4. 预导入：在测试环境执行导入，验证表结构和约束
5. 正式导入：在生产环境维护窗口（10月12日 2:00-6:00）执行
6. 校验：导入后运行数据比对脚本，验证数量一致性

五、注意事项
-----------

- 迁移前必须对旧系统数据库做完整备份
- 导入期间 PostgreSQL 主库暂停对外服务
- 密码字段全部重置为随机密码，用户首次登录需通过邮件重置
- 迁移失败时保留原始 CSV 文件，可重新导入
- 财务相关数据迁移后需财务部门人工抽检确认
