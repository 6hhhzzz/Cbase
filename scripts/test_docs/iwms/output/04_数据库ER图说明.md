## 一、数据库概述

IWMS 数据库按微服务边界划分为 6 个 Schema，共 28 张业务表。本文档说明核心表结构和表间关系。

## 二、核心表 ER 关系

### 2.1 入库域 (wms_inbound)

```
inbound_orders (入库单)
  ├── id: UUID PK
  ├── order_no: VARCHAR(32) UNIQUE   -- 如 IN-20260315-0001
  ├── type: VARCHAR(16)              -- purchase/return/transfer
  ├── supplier_id: UUID FK           -- 供应商
  ├── warehouse_id: UUID FK          -- 仓库
  ├── status: VARCHAR(16)            -- draft/confirmed/quality_check/complete
  ├── total_quantity: INTEGER
  ├── created_by: UUID FK → users.id
  └── created_at: TIMESTAMP

inbound_items (入库明细)
  ├── id: UUID PK
  ├── inbound_order_id: UUID FK → inbound_orders.id
  ├── sku_id: UUID FK               -- 商品 SKU
  ├── expected_qty: INTEGER          -- 预期数量
  ├── actual_qty: INTEGER            -- 实收数量
  ├── batch_no: VARCHAR(32)          -- 批次号
  ├── location_id: UUID FK           -- 上架货位
  └── quality_result: VARCHAR(8)     -- pass/fail
```

### 2.2 出库域 (wms_outbound)

```
outbound_orders (出库单)
  ├── id: UUID PK
  ├── order_no: VARCHAR(32) UNIQUE
  ├── type: VARCHAR(16)              -- sales/transfer
  ├── erp_order_id: VARCHAR(64)      -- 关联 ERP 订单
  ├── customer_name: VARCHAR(64)
  ├── express_company: VARCHAR(16)   -- 快递公司
  ├── express_no: VARCHAR(32)        -- 快递单号
  ├── priority: VARCHAR(8)           -- normal/urgent/emergency
  ├── status: VARCHAR(16)            -- pending/picking/packing/shipped
  ├── total_amount: DECIMAL(12,2)    -- 订单金额
  ├── approval_status: VARCHAR(16)   -- pending/approved/rejected
  ├── created_by: UUID FK → users.id
  └── created_at: TIMESTAMP

outbound_items (出库明细)
  ├── id: UUID PK
  ├── outbound_order_id: UUID FK → outbound_orders.id
  ├── sku_id: UUID FK
  ├── quantity: INTEGER
  ├── picked_qty: INTEGER            -- 已拣数量
  └── batch_no: VARCHAR(32)          -- 出库批次（用于追溯）
```

### 2.3 库存域 (wms_inventory)

```
inventory (实时库存)
  ├── id: UUID PK
  ├── sku_id: UUID FK               -- 唯一，每种商品一条记录
  ├── warehouse_id: UUID FK
  ├── location_id: UUID FK
  ├── quantity: INTEGER              -- 当前库存量
  ├── locked_quantity: INTEGER       -- 锁定库存（已分配未出库）
  ├── safety_stock: INTEGER          -- 安全库存阈值
  ├── last_updated: TIMESTAMP
  └── UNIQUE(sku_id, warehouse_id, location_id)

inventory_log (库存变更日志)
  ├── id: UUID PK
  ├── sku_id: UUID FK
  ├── change_type: VARCHAR(16)       -- inbound/outbound/adjust/lock/unlock
  ├── change_qty: INTEGER            -- 变更量（正数为增，负数为减）
  ├── before_qty: INTEGER            -- 变更前数量
  ├── after_qty: INTEGER             -- 变更后数量
  ├── related_order_no: VARCHAR(32)  -- 关联单据号
  ├── created_by: UUID FK → users.id
  └── created_at: TIMESTAMP
```

### 2.4 权限域 (wms_auth)

```
users (用户表)
  ├── id: UUID PK
  ├── username: VARCHAR(32) UNIQUE
  ├── password: VARCHAR(128)         -- BCrypt 加密
  ├── display_name: VARCHAR(32)
  ├── phone: VARCHAR(16)             -- 手机号
  ├── email: VARCHAR(64)
  ├── status: VARCHAR(8)             -- active/disabled
  └── created_at: TIMESTAMP

roles (角色表)
  ├── id: UUID PK
  ├── name: VARCHAR(32) UNIQUE       -- super_admin/warehouse_manager/supervisor/operator/inspector/finance/viewer
  └── description: VARCHAR(128)

permissions (权限表)
  ├── id: UUID PK
  ├── role_id: UUID FK → roles.id
  ├── resource: VARCHAR(64)          -- 资源路径，如 inbound:write
  └── action: VARCHAR(16)            -- read/write/delete/approve

audit_logs (审计日志)
  ├── id: UUID PK
  ├── user_id: UUID FK → users.id
  ├── action: VARCHAR(32)            -- 操作类型
  ├── target_type: VARCHAR(32)       -- 操作对象类型
  ├── target_id: UUID                -- 操作对象 ID
  ├── details: JSONB                 -- 操作详情
  └── created_at: TIMESTAMP

user_roles (用户角色关联)
  ├── user_id: UUID FK → users.id
  ├── role_id: UUID FK → roles.id
  └── PRIMARY KEY(user_id, role_id)
```

## 三、关键索引设计

| 表 | 索引 | 用途 |
|----|------|------|
| inventory | idx_inv_sku_warehouse (sku_id, warehouse_id) | 库存查询核心索引 |
| inventory_log | idx_invlog_created (created_at) | 按时段查询变更记录 |
| inbound_orders | idx_inbound_status (status) | 按状态筛选 |
| outbound_orders | idx_outbound_status (status) | 按状态筛选 |
| outbound_orders | idx_outbound_approval (approval_status) | 审批列表查询 |
| audit_logs | idx_audit_user (user_id, created_at) | 按操作人+时间查询 |
| inbound_items | idx_inbound_batch (batch_no) | 批次追溯 |
| outbound_items | idx_outbound_batch (batch_no) | 出库批次追溯 |

## 四、数据迁移表映射

旧系统（Delphi + SQL Server 2008）到新系统（PostgreSQL 16）的表映射关系见《数据迁移方案》文档。关键映射包括：

| 旧表名 | 新表名 | 说明 |
|--------|-------|------|
| T_Goods | inventory (sku_id) | 商品主数据 |
| T_Stock | inventory (quantity) | 库存数量 |
| T_In_Master / T_In_Detail | inbound_orders / inbound_items | 入库单拆分 |
| T_Out_Master / T_Out_Detail | outbound_orders / outbound_items | 出库单拆分 |
| T_Users | users | 用户表，密码需重新哈希 |
