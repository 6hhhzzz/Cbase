## 一、引言

### 1.1 编写目的

本文档定义 IWMS 系统的功能需求和非功能需求，作为开发、测试和验收的依据。目标读者包括开发团队、测试团队、客户方项目组成员。

### 1.2 适用范围

本文档涵盖 IWMS 全部 6 个功能模块：入库管理、出库管理、库存管理、权限管理、报表中心、系统管理。

## 二、功能性需求

### 2.1 入库管理模块 (P0)

#### 2.1.1 采购入库

- **REQ-IN-001**：仓管员可通过 PC 端或 PDA 端创建采购入库单
- **REQ-IN-002**：系统支持扫描或手动输入采购单号，自动关联 ERP 采购订单数据
- **REQ-IN-003**：入库时自动生成批次号，规则为 `IN-YYYYMMDD-流水号`（如 IN-20260315-0001）
- **REQ-IN-004**：支持质检确认环节，质检员扫码确认后方可完成上架
- **REQ-IN-005**：入库完成后，系统自动更新库存数量，并向 ERP 系统回传入库确认信息

#### 2.1.2 退货入库

- **REQ-IN-006**：支持根据原出库单号发起退货入库流程
- **REQ-IN-007**：退货入库需经质检判定，区分"良品入库"和"残次品入库"
- **REQ-IN-008**：残次品入库后进入锁定库存，不可直接销售

#### 2.1.3 调拨入库

- **REQ-IN-009**：支持接收来自其他仓库的调拨入库
- **REQ-IN-010**：调拨入库需匹配调拨出库单，系统自动校验数量一致性

### 2.2 出库管理模块 (P0)

#### 2.2.1 销售出库

- **REQ-OUT-001**：与 ERP 系统同步销售订单，自动生成出库任务
- **REQ-OUT-002**：支持波次拣货策略：按快递公司、按区域、按订单优先级（普通/加急/紧急）分组
- **REQ-OUT-003**：PDA 端显示拣货路径（按货位排序），引导仓管员按最优路径行走
- **REQ-OUT-004**：支持复核打包环节，扫描商品条码验证出库商品正确性
- **REQ-OUT-005**：自动打印快递面单，支持顺丰、中通、圆通、韵达、EMS 五家快递公司

#### 2.2.2 出库审批规则

| 订单金额 | 审批层级 | 备注 |
|---------|---------|------|
| ≤ 5,000 元 | 无需审批 | 系统自动放行 |
| 5,000 - 50,000 元 | 仓库主管审批 | |
| 50,000 - 200,000 元 | 仓库主管 + 财务审批 | |
| > 200,000 元 | 仓库主管 + 财务 + 总经理审批 | 需附合同复印件 |

### 2.3 库存管理模块 (P0)

- **REQ-INV-001**：实时库存查询，支持按 SKU、货位、批次、入库时间等多维度筛选
- **REQ-INV-002**：库存预警：当库存数量低于安全库存阈值时，系统自动发送通知给采购部门
- **REQ-INV-003**：支持循环盘点（Cycle Count）和全面盘点两种模式
- **REQ-INV-004**：盘点差异自动生成盘盈/盘亏单据，经主管审批后调整库存
- **REQ-INV-005**：批次追溯：可根据批次号正向追踪出库去向，或根据销售单号反向追溯到入库批次
- **REQ-INV-006**：库存数据每 5 分钟同步一次至 ERP 系统

### 2.4 权限管理模块 (P1)

- **REQ-AUTH-001**：采用 RBAC（基于角色的访问控制）模型
- **REQ-AUTH-002**：系统预置角色：超级管理员、仓库经理、仓库主管、仓管员、质检员、财务、只读用户
- **REQ-AUTH-003**：支持数据级权限隔离：仓管员只能查看自己所负责仓库区域的数据
- **REQ-AUTH-004**：所有敏感操作（删除、修改库存、审批）需二次确认，并记录审计日志
- **REQ-AUTH-005**：登录失败超过 5 次，账号锁定 30 分钟

详细权限矩阵见《权限模块设计文档》。

### 2.5 报表中心 (P1)

- **REQ-RPT-001**：库存周转率分析报表（支持按日/周/月/季度维度）
- **REQ-RPT-002**：出入库量统计（按 SKU、按仓库区域、按时间段）
- **REQ-RPT-003**：拣货效率报表（人均拣货件数/小时、拣货准确率）
- **REQ-RPT-004**：呆滞库存分析（超过 90 天未出库的商品）
- **REQ-RPT-005**：所有报表支持导出为 Excel 和 PDF 格式

### 2.6 系统管理 (P1)

- **REQ-SYS-001**：用户管理：创建、编辑、禁用、删除用户
- **REQ-SYS-002**：操作日志审计：记录所有用户的关键操作，保留期不少于 12 个月
- **REQ-SYS-003**：系统配置管理：支持配置快递公司、打印机、仓库区域等参数
- **REQ-SYS-004**：数据备份：每日凌晨 2:00 自动全量备份，备份文件保留最近 30 天

## 三、非功能性需求

### 3.1 性能要求

- 页面加载时间 ≤ 2 秒（99 分位）
- API 响应时间 ≤ 500ms（平均值），≤ 2s（99 分位）
- PDA 端扫码识别到数据显示 ≤ 1 秒
- 系统并发用户数 ≥ 200

### 3.2 可用性

- 系统可用性 ≥ 99.9%（月度），计划内停机时间窗口：每周日凌晨 2:00-4:00
- 关键业务模块（入库、出库）支持离线模式：PDA 端在网络中断时可暂存数据，网络恢复后自动同步

### 3.3 安全性

- 所有 API 端点需通过 JWT Token 认证
- 敏感数据传输使用 HTTPS 加密
- 用户密码使用 BCrypt 加密存储
- 生产环境数据与测试环境数据完全隔离

### 3.4 兼容性

- 浏览器：Chrome 90+, Edge 90+, Firefox 88+
- PDA 端：Android 10+ 系统，适配主流 PDA 品牌（斑马、霍尼韦尔、优博讯）
- ERP 对接：用友 U8+ v13.0 API

## 四、数据迁移需求

客户现有系统中约有 8 年的历史数据需要迁移：

- 商品主数据：约 12,000 条 SKU
- 库存数据：约 50,000 条实时库存记录
- 历史出入库单：约 300,000 条
- 数据清洗规则见《数据迁移方案》
