## 一、项目背景

速达物流集团（以下简称"客户"）现有仓储管理系统上线已超过 8 年，基于 Delphi + SQL Server 2008 架构，存在以下突出问题：

1. **性能瓶颈**：日处理订单量超过 5000 单时，系统响应时间从 2 秒下降至 15 秒以上，严重影响出库效率
2. **功能缺失**：不支持移动端扫码入库、实时库存可视化、智能波次拣货等现代仓储管理功能
3. **维护困难**：原开发团队已解散，源代码文档缺失，修改现有功能风险极高
4. **集成能力弱**：无法与企业现有的 ERP 系统（用友 U8+）和 TMS 运输管理系统对接

客户于 2026 年 1 月公开招标，ACME 科技以技术方案评分第一名中标。

## 二、项目目标

### 2.1 核心目标

- 替代旧仓储管理系统，实现全业务流程线上化
- 日均处理订单能力提升至 20,000 单以上
- 系统平均响应时间不超过 2 秒（99 分位）
- 与客户现有 ERP、TMS 系统完成数据对接

### 2.2 功能模块

| 模块 | 功能描述 | 优先级 |
|------|---------|--------|
| 入库管理 | 采购入库、退货入库、调拨入库，支持 PDA 扫码 | P0 |
| 出库管理 | 销售出库、波次拣货、复核打包、快递面单打印 | P0 |
| 库存管理 | 实时库存查询、库存预警、盘点管理、批次追溯 | P0 |
| 权限管理 | 基于 RBAC 的角色权限控制，支持数据级隔离 | P1 |
| 报表中心 | 库存周转分析、出入库统计、拣货效率报表 | P1 |
| 系统管理 | 用户管理、日志审计、系统配置、数据备份 | P1 |

## 三、技术方案概要

总体采用 B/S 架构，前后端分离：

- **后端**：Java 17 + Spring Boot 3.3，微服务架构
- **前端**：Vue 3 + Element Plus
- **数据库**：PostgreSQL 16 + Redis 7 缓存
- **消息队列**：RabbitMQ 3，处理订单异步流转
- **对象存储**：MinIO，存放电子签收单扫描件
- **部署**：Docker Compose + Kubernetes（客户方提供集群）

详细架构设计见《系统架构设计文档》。
