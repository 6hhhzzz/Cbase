# IWMS 前端组件设计规范

## 文档信息

- **作者**: 刘洋（前端组长）
- **日期**: 2026 年 4 月 25 日
- **技术栈**: Vue 3.4 + Element Plus 2.6 + Pinia 2.1 + Vite 5

## 一、设计原则

1. **组件化优先**：可复用 UI 片段抽象为独立组件，避免重复代码
2. **状态统一管理**：跨组件共享的状态使用 Pinia Store，组件内私有状态用 ref/reactive
3. **移动端适配**：PDA 端使用同一套代码库，通过路由级布局切换实现响应式
4. **类型安全**：全面使用 TypeScript，所有 Props/Emits/Store 有类型定义

## 二、路由设计

```
/                         → 布局：登录页
/login                    → 登录
/                         → 布局：主框架（侧边栏+顶栏）
  /dashboard              → 首页看板
  /inbound                → 入库管理
    /inbound/orders       → 入库单列表
    /inbound/orders/:id   → 入库单详情
    /inbound/create       → 创建入库单
  /outbound               → 出库管理
    /outbound/orders      → 出库单列表
    /outbound/waves       → 波次拣货
    /outbound/packing     → 复核打包
  /inventory              → 库存管理
    /inventory/query      → 库存查询
    /inventory/count      → 盘点管理
  /report                 → 报表中心
  /system                 → 系统管理
    /system/users         → 用户管理
    /system/roles         → 角色管理
    /system/audit         → 审计日志
  /pda                    → PDA 布局（简化侧边栏）
    /pda/inbound          → PDA 入库
    /pda/outbound         → PDA 出库
    /pda/inventory        → PDA 盘点
```

## 四、状态管理 (Pinia Store)

| Store | 路径 | 状态 |
|-------|------|------|
| useAuthStore | stores/auth.ts | Token、用户信息、权限列表 |
| useInboundStore | stores/inbound.ts | 入库单列表、当前编辑的入库单 |
| useOutboundStore | stores/outbound.ts | 出库单列表、波次任务 |
| useInventoryStore | stores/inventory.ts | 库存查询缓存 |
| useAppStore | stores/app.ts | 侧边栏折叠、当前仓库选择 |

## 五、PDA 端适配

PDA 端核心差异：
- 布局：单栏无侧边栏，顶栏仅显示当前功能标题
- 字体：最小 14px（确保扫描可读性）
- 按钮：最小触控区域 44×44px（H5 标准）
- 扫码：使用 PDA 厂商提供的 JSBridge 调用摄像头扫码
- 离线：使用 localStorage 暂存操作队列，网络恢复后批量提交

## 六、样式规范

- 主色调：#1890ff（Element Plus 默认蓝）
- 成功：#52c41a  /  警告：#faad14  /  危险：#ff4d4f
- 字体：优先使用系统字体栈 `-apple-system, 'Microsoft YaHei', sans-serif`
- 间距：以 4px 为基数，常用间距 8px、12px、16px、24px
- 所有自定义样式使用 scoped 作用域
