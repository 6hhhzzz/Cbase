## 三、快速启动（测试环境）

```bash
# 1. 克隆代码
git clone git@acme.com:iwms/iwms-platform.git
cd iwms-platform

# 2. 启动基础设施
docker compose up -d postgres redis rabbitmq minio

# 3. 初始化数据库
docker compose exec postgres psql -U iwms -d iwms_db \
  -f /docker-entrypoint-initdb.d/init.sql

# 4. 启动后端服务
cd backend && mvn spring-boot:run -Dspring.profiles.active=test

# 5. 启动前端
cd frontend && npm install && npm run dev
```

## 五、K8s 部署（生产环境）

```yaml
# wms-api deployment 示例
apiVersion: apps/v1
kind: Deployment
metadata:
  name: wms-api
spec:
  replicas: 2
  selector:
    matchLabels:
      app: wms-api
  template:
    metadata:
      labels:
        app: wms-api
    spec:
      containers:
      - name: wms-api
        image: registry.acme.com/iwms/api:1.0.0
        ports:
        - containerPort: 8080
        envFrom:
        - secretRef:
            name: iwms-secrets
        resources:
          requests:
            memory: "512Mi"
            cpu: "500m"
          limits:
            memory: "2Gi"
            cpu: "2000m"
```

## 七、监控与告警

- **健康检查**: `GET /actuator/health`，K8s liveness probe 每 30 秒探测
- **日志**: JSON 格式输出到 stdout，由 Fluentd 收集至 Elasticsearch
- **指标**: Prometheus + Grafana，关键指标包括 API 响应时间、错误率、JVM 内存
- **告警规则**: API 错误率 > 5% (critical)、数据库连接池耗尽 (critical)、磁盘使用率 > 85% (warning)

## 八、备份策略

| 数据 | 备份频率 | 保留期 | 方式 |
|------|---------|--------|------|
| PostgreSQL | 每日凌晨 2:00 全量 | 30 天 | pg_dump + 压缩上传至 MinIO |
| Redis | 每小时 RDB 快照 | 7 天 | 自动 dump.rdb |
| MinIO | 每日 | 30 天 | mc mirror 至备份 Bucket |
