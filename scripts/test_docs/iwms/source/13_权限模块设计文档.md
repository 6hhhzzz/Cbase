# 权限模块设计文档

## 文档信息

- **作者**: 陈强（后端组长）
- **日期**: 2026 年 6 月 10 日
- **技术依赖**: Spring Security + JWT + Redis

## 一、概述

IWMS 权限模块采用 **RBAC（基于角色的访问控制）** 模型。系统预置 7 种角色，每个角色分配一组权限（Permission），用户通过关联角色获得权限。

## 二、角色定义

| 角色 | 标识 | 说明 | 典型用户 |
|------|------|------|---------|
| 超级管理员 | super_admin | 拥有系统全部权限，可管理用户和角色 | 李明辉（技术负责人） |
| 仓库经理 | warehouse_manager | 管理仓库运营数据，审批关键操作 | 张伟（项目经理） |
| 仓库主管 | supervisor | 审批日常出库，管理下属仓管员 | 客户方仓库主管 |
| 仓管员 | operator | 执行入库、出库、盘点等日常操作 | 仓库一线员工 |
| 质检员 | inspector | 入库质检、退货判定 | 客户方质检员 |
| 财务 | finance | 查看报表、审批大额出库 | 客户方财务 |
| 只读用户 | viewer | 仅查看数据，不可操作 | 客户方审计人员 |

## 四、数据级权限隔离

除功能权限外，部分角色有**数据级权限**限制：

| 角色 | 数据隔离规则 |
|------|------------|
| supervisor | 只能管理自己负责的仓库区域（warehouse_zone） |
| operator | 只能查看自己创建的入库/出库单，以及本仓库区域的库存 |
| inspector | 只能查看分配给自己的质检任务 |

实现方式：API 查询时自动注入 `WHERE warehouse_zone_id = $current_user_zone` 条件。

## 五、认证流程

### 5.1 JWT 双 Token 机制

```
POST /api/v1/auth/login
  → 返回 {access_token (30min), refresh_token (7d)}

后续 API 请求：
  Authorization: Bearer <access_token>

Access Token 过期：
  POST /api/v1/auth/refresh {refresh_token}
    → 返回新的 access_token
```

### 5.2 Token 结构

```json
{
  "sub": "user-uuid",
  "username": "zhangwei",
  "role": "warehouse_manager",
  "permissions": [
    "inbound:read", "inbound:write",
    "outbound:read", "outbound:approve",
    "inventory:read", "inventory:write",
    "report:read", "report:export",
    "system:read"
  ],
  "iat": 1715900000,
  "exp": 1715901800
}
```

### 5.3 安全规则

- 密码使用 BCrypt（cost=10）加密存储
- 登录失败 ≥ 5 次，账号锁定 30 分钟
- 所有敏感操作写入审计日志（audit_logs 表）
- 审计日志保留期 12 个月，包含操作人、操作时间、操作类型、IP 地址、User-Agent

## 六、与 API 接口的关系

详细的 API 端点与权限对应关系见《REST API 接口规范》文档。关键对应：

| API 端点 | 所需权限 | 说明 |
|---------|---------|------|
| POST /inbound/orders | inbound:write | 创建入库单 |
| POST /outbound/orders/{id}/approve | outbound:approve | 审批出库单 |
| PUT /inventory/count/{id} | inventory:write | 提交盘点结果 |
| DELETE /auth/users/{id} | auth:write | 禁用用户（仅 super_admin） |
| GET /reports/export/{type} | report:export | 导出报表 |

## 七、数据库表

权限模块使用 5 张表：`users`、`user_roles`、`roles`、`permissions`、`audit_logs`。

详细表结构和索引见《数据库 ER 图说明》文档的 2.4 节。
